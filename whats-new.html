v2.11.0 Latest April 2026
🏢
Org ID Display — now shows Record ID in Setup
On Setup pages that open a specific record — such as a user, permission set, or messaging channel — the Setup header now shows that Record ID instead of the Org ID. The Org ID still appears on the Setup homepage and on pages with no record context. Updates automatically as you navigate.
Feature

🎉
What's New page & Feature Tour
When the extension updates, a badge appears on the toolbar icon and this page opens automatically. A full Feature Tour with screenshots for all 15 features is accessible from the options page and this page.
Feature
v2.9.4 April 2026
↩️
App Switcher Return — rewritten
Fixed App Switcher Return not working. The previous implementation relied on shadow DOM traversal and History API patching — neither of which Salesforce's LWC router uses. Rewrote to listen directly on app launcher links, saving the current URL at click time so the return always lands on the right record.
Fix
v2.9.3 April 2026
🔍
User Search — Lightning fix
Fixed User Search returning no results on Lightning pages when Setup hadn't been visited first. The extension was using the wrong session cookie domain — Salesforce's REST API only accepts the my.salesforce.com session. Now always tries the correct API domain first.
Fix
v2.9.0 – 2.9.2 April 2026
🔍
User Search
Search for any user by name, username, or email — including deactivated users Salesforce's built-in search ignores. Results include an active/inactive badge and link directly to the user record. Available in both Lightning and Setup.
Feature
🛡️
Stability improvements
Features now fail in isolation — if one encounters an error it no longer affects others. Guards added against invalidated extension context.
Fix
📌
User Search tooltip clipping
Fixed tooltip clipping at the top of the viewport when the search button appears near the top of the page.
